## Goal at session start

Add an optional 3-round deliberation mode to the committee, driven as a re-vote
over fixed candidates, so we can see whether/how the tally moves across rounds —
without disturbing any v1/v2/v3 path. Source of truth: `docs/v4-design.md`.

## What I did

Implemented exactly the four edits the design doc scopes, all additive:

- **`src/agents/committee.py`** — `cast_vote` and `_format_user_message` gained an
  optional `debate_context: str | None = None`. When set, the peer-positions block
  is appended *after* the PROMPT/CANDIDATE block (never substituted), so a persona
  always re-reads exactly what it is judging. `None` (every existing caller:
  pipeline, batch, default revote, round 1) → the message is byte-for-byte unchanged.

- **`src/agents/debate.py`** (new) — `DebateRound`, `DebateResult`, `run_debate(...)`,
  `_build_debate_context(...)`. Round 1 is the standard independent vote (empty
  context); rounds ≥2 show each persona every *other* member's previous-round
  vote+reason (by role name + leaning) plus its own prior vote, then it may revise.
  Personas vote concurrently within a round (`asyncio.gather`); rounds run
  sequentially; `adjudicate` runs after each round; round N is final. Reuses
  `Vote`/`cast_vote`, `adjudicate`/`Decision`, `REFUSAL_MESSAGE` unchanged.

- **`src/revote.py`** — debate is a **program argument**, not a new script. New
  opt-in flags: `--debate-rounds N` (default 1), `--contested` (swing band
  approves ∈ {⌊n/2⌋, ⌊n/2⌋+1} = {3,4} for the 7-member committee), `--controls N`
  (default 0; clearly-decided extremes, placed first so they survive a tight
  `--limit`), `--prompt-ids` (comma list or `@file`). The only change to existing
  logic is one branch in `_revote_one`: `if debate_rounds >= 2: run_debate(...)`.
  Output is additive — top-level `votes`/`decision` stay the final-round result so
  `analyze_v2` reads it unchanged; per-round development under a new `debate` key.
  Console prints the development: `R1 A=4/B=3 RELEASED → R2 … → R3 …  flip@R2`.

- **`notebooks/v3_analysis.ipynb`** — appended a light v4 block (1 markdown + 3 code
  cells): per-round tally (mean approves + release-rate R1→R2→R3), outcome flips,
  per-persona flip tally with direction, and R3-vs-R1-vs-`ground_truth` on the
  subset. Reads `data/v3/results/qwen3-balanced-7-pr-debate-v3.jsonl`, falling back
  to `/tmp/debate-smoke.jsonl`.

Nothing touched in `configs/`, `data/`, personas, `pipeline.py`, `batch.py`,
`adjudicator.py`, or `run.py`. All new flags default to current behavior.
