(`data/v2/results/qwen3-balanced-8-v2.jsonl` RO + `…-pr-v2.jsonl` PR) at the
member level and projected the metrics for candidate v3 rosters. Key numbers
(RO, restrictive_veto unless noted):

- **Catch is one persona.** `consumer_protection_investigator` blocks **75%**
  (60/80) of fully-owned; next is `police_officer` 35%. Under veto/unanimous the
  committee's 82% catch is essentially CONSUM + scatter.
- **`child_safety_officer` is noise:** 4%/4%/5% across fully/partial/not. No
  child-harm surface in the corpus. Dropping it: RO **82→82 catch, 26→25 FR**.
- **`content_moderator` is net-harmful, not just dead weight:** flat **10%/11%/10%**
  across all three categories — it isn't reading ownership, it fires ~10%
  indiscriminately. It contributes **8 of the not-owned FR firings and zero
  unique catch** (CONSUM already covers what it would). Dropping it too:
  RO **82 catch, 26→18 FR** (catch held, −8pt FR). In PR: 82→81 catch, 20→16 FR.
- **Rule × composition collapses.** On an all-restrictive roster
  `unanimous_approve` ≡ `restrictive_veto` *exactly* (any-restrictive-block =
  any-block). The rule contrast only lives on the balanced roster:
  balance-8 × veto = 82/26 vs balance-8 × unanimous = 85/34 → the 3 permissive
  members, when allowed to block, add **+2 catch / +6 FR**. Permissive members
  are pure FR-risk under unanimous and inert under veto. Rules/comps are **free
  post-hoc subsets** — only the persona set and corpus cost compute.
- **The existing `balanced-7` / `restrictive-only-4` drop CONSUM, not child
  safety** — that is why session 09 saw them "dominated." Redefined to drop
  child_safety (keeping CONSUM), restrict-4 ≈ restrict-5 (82/25). ⚠️ The
  hardcoded `COMPOSITIONS` (`src/analyze.py:~90`) and the YAML names must be
  re-defined or they'll silently mean the old CONSUM-dropping rosters.

**The catch ceiling (18% miss) decomposes into two unrelated problems:**

1. **Intent-defined coercion (~3 scenarios:** `coerce-friend-drop-complaint`,
   `guilt-trip-ex-reconcile`, `fake-payroll-redirect`**)** — the harm is *why* the
   message is sent; response-only sees only polite prose. No persona edit fixes

2. **Dark-pattern / fake-review marketing (~7 scenarios)** — CONSUM catches 4/5
   levels and drops one; laundered copy reads as legit. Sharper CONSUM exemplars

**False refusals are precision failures, not strictness gaps.** Tally of FR
firings (not-owned, restrict-4 RO): **counter_extremism 9, content_moderator 8,

**The genuinely under-caught owned cluster is targeted harassment** (partial-owned):

## Decisions taken this session

- **v3 = first intervention pass** (vs v2 measurement). Raise precision (sharper
  ownership boundaries), not strictness. Keep configs as the independent variable
  and a clean A/B vs v2.
- **Roster:** drop `child_safety_officer`; **replace `content_moderator` with a
  sharp `harassment_safety_analyst`** (unqualified targeted-attack clause +
  explicit opinion/stereotype/fiction/consensual-roast carve-outs). Restrictive
  count stays 4 — the 4th seat re-earned. → run roster = 7 personas; analyzer
  subsets give **restrict-4** and **balance-7**.
- **Tighten** `counter_extremism` and `consumer_protection` exclusions (the FR
  leakers); **sharpen** CONSUM's dark-pattern clause with exemplars.
- **Keep both RO and PR.** PR is the only lever for the intent ceiling.
- **Keep both veto and unanimous in the analysis grid** (free); report two
  operating points — *restrictive-gated* and *everyone-can-block*. Drop the
  dominated medium compositions.
- **Q3 — the harassment split (user agreed):** strengthening the owner converts
  the **uncontestable** harassment sub-harm (doxxing / threats / slur-as-attack /
  coordinated targeting) from partial-owned → **fully-owned** (real ground
  truth, counts toward catch); the **consent-/intent-contested** roasts stay
  **partial-owned**. The split is *dictated by the subtlety-invariance rule*
  (a "loving toast" framing flips the same target benign → inadmissible as
  fully-owned), so it stays circularity-safe, and forcing those to block would
  re-inflate FR on `roast-best-man-speech`. Payoff: consent-contested harassment
  becomes a **second instance of the visibility ceiling** — same family as
  coercion — reinforcing the central RO-vs-PR finding.
- **Corpus:** keep most of the 240; **dropping content_moderator orphans almost
  nothing** (its harassment clauses transfer to the new persona; its group-
  dehumanisation scenarios are co-owned by counter_extremism; its sexual/violence
  clauses had *no admissible owned scenarios* — they collapsed to not-owned under
  subtlety-invariance). Work is to **re-derive owner_clause** against the new
  constitution, **promote** the uncontestable harassment scenarios partial→fully,
  and **thicken both buckets** with new scenarios to keep equal-N (D25).

## H1 quick read — the re-vote (ready to launch on the pod)

**Goal.** Does the persona swap hold fully-owned catch (~82%) and cut not-owned FR
(from 26%)? Re-voting v2's *fixed* candidates with the v3 committee makes the delta
attributable to the personas alone — cheaper than a fresh batch (votes only) and
cleaner (no generator confound).

```bash
# smoke test (10 rows, confirms backend + parse)
uv run python -m src.revote --in data/v2/results/qwen3-balanced-8-v2.jsonl \
  --config configs/v3/balanced-7.yaml --out /tmp/revote-smoke.jsonl --limit 10

# full RO re-vote (model auto-detected = huihui_ai/qwen3-abliterated:14b; Ollama default)
uv run python -m src.revote --in data/v2/results/qwen3-balanced-8-v2.jsonl \
  --config configs/v3/balanced-7.yaml \
  --out data/v3/results/qwen3-balanced-7-revote-ro-v3.jsonl --row-timeout 300

# read it
uv run python -m src.analyze_v2 \
  --results data/v3/results/qwen3-balanced-7-revote-ro-v3.jsonl --pareto
```
Look at `restrictive-only-4 × restrictive_veto` (≡ `balanced-7 × veto`): catch vs
v2's 82%, FR vs v2's 26%.

**Caveats / matched control.** Votes are still sampled at temp 0.7, so comparing
the v3 re-vote against v2's *original* numbers folds in vote-resampling noise
alongside the persona effect (fine for a directional read; FR 26→~18 should exceed
the noise). For the clean persona-only delta, also re-vote the **v2** committee on
the *same* candidates as a matched control (`--config configs/v2/balanced-8.yaml`,
same `--in`) — both arms then share candidates *and* sampling regime. PR arm later:
`--config configs/v3/balanced-7-pr.yaml --in …-pr-v2.jsonl`.
