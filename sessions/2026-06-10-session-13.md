## Goal at session start

Produce a v3 analysis notebook equivalent to `notebooks/v2_analysis.ipynb` so the
two versions can be compared cell-for-cell — pointed at the new v3 corpus and both
v3 arms (`data/v3/results/qwen3-balanced-7{,-pr}-v3.jsonl`).

## What blocked it

`src/plot_v2.py` was v2-hardcoded: the static v2 `COMPOSITIONS`, `PERSONA_ORDER`,
`REMIT_OWNER`, and a v2-only `owner_clause` parser. v3 changed the roster (D29:
dropped `child_safety_officer`, `content_moderator` → `harassment_safety_analyst`)
**and reused the names** `balanced-7` / `restrictive-only-4` / `balanced-3` to mean
different rosters, so running the v2 plot engine on v3 votes silently drops every
composition (the len-mismatch guard) and the persona/remit figures come out empty or
all-NaN. v3's `owner_clause` also names the owning *persona* directly
(`consumer_protection_investigator: …`) instead of v2's short codes (`CONSUM`).

## What I did

**Made `plot_v2.py` version-aware** — it now auto-routes the roster from the data it
is handed, the same `harassment_safety_analyst`-marker trick `src.analyze_v2` uses
(`compositions_for(rows)`), so the one module serves both corpora and v2 behaviour is
byte-identical:

- `load_all` resolves comps per arm via `compositions_for` (was the static dict).
- Added `_comps_for` / `_resolve_comp` / `_persona_order` helpers; the v2 default
  `balanced-8` degrades to v3 `balanced-7` instead of `KeyError`.
- `remit_curve_df`, `_decision_at` (Fig 5), `scenario_decision_df` (Fig 8) resolve
  comps from the rows rather than `COMPOSITIONS[comp]`.
- Persona figures (6 / 6b / 7) derive roster order from the data; Pareto figures
  (2 / 2b-zoom) build the composition→marker map + legend from the comps present.
- `load_remit_map` now globs `{cat}-owned-*.jsonl` and the parser accepts both v2
  UPPER-CASE codes and v3 snake_case persona keys.
- Added the v3 remit taxonomy (`_V3_REMIT_ORDER/LABEL`, `_remit_cfg`): CONSUM /
  POLICE / CTEX / **HARASS** / EDU + `none`, each owning itself (owner == persona).
  (Caught `educator` — it still owns the falsehood-as-fact partials, as in v2's EDU.)

**Created `notebooks/v3_analysis.ipynb`** — mirrors the v2 notebook cell-for-cell
(same 13 figure calls, same three-granularity narrative) but with v3 `ARMS` paths,
`COMP='balanced-7'`, the `balanced-7 × restrictive_veto` operating point, the
`../data/v3/prompts` remit dir, and v3 prose tying each figure to H1–H4 + the
harassment cluster. Built and executed headlessly via nbconvert.
